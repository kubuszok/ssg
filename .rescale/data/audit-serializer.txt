NOTES
===============================================================================

1. SSG uses a "format to String then append" pattern (formatValue, formatColor,
   formatList, etc.) instead of dart-sass's direct-to-buffer visitor pattern.
   This is a design difference but functionally equivalent for correctness.
   The buffer approach is more memory-efficient for large stylesheets.

2. SSG's selector serialization bypasses the visitor entirely, relying on
   CompoundSelector.toString() and a custom writeComplexSelectorTo helper.
   This works for most cases but cannot implement:
   - :not(%invisible) -> * optimization
   - Empty compound -> * fallback
   - Attribute selector value quoting with identifier detection
   - PUA escaping in selector strings

3. The SSG source map implementation is simpler (records per-declaration
   rather than per-span) but produces valid v3 source maps.

4. The _quote field (controlling whether quoted strings emit quotes) is
   missing from the SSG constructor. This affects the serializeValue entry
   point when called with quote=false (used by string interpolation).

5. SassCalculation, SassBoolean, SassFunction, and SassMixin value types
   may not exist yet in SSG. The missing visitor methods for these types
   will need the corresponding value types to be ported first.

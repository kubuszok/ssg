SUBSYSTEM PRIORITIES (ordered by spec impact)
================================================================================

PRIORITY 1: PARSER — Expression parser unification
  The text-based _expression() fallback is the root cause of hundreds of
  failures. The partial RD parser (_rdExpression) needs to be completed to
  handle ALL expression forms and wired as the sole expression parser.
  This unlocks: bracket lists, unicode ranges, selector expressions,
  modern if(), proper interpolation, correct operator precedence.
  Estimated: 450 lines, ~500-800 spec fixes.

PRIORITY 2: EVALUATOR — Module orchestration (_execute, _combineCss, _loadModule)
  The ad-hoc module loading in visitUseRule/visitForwardRule lacks proper
  environment save/restore, configuration assertion, module caching, and
  CSS combination. This causes cross-module state pollution, incorrect
  output ordering, and missing configuration validation.
  Estimated: 340 lines, ~100-200 spec fixes.

PRIORITY 3: SERIALIZER — Faithful port of serialize.dart
  Missing calculation serialization, full color dispatch, selector visitor
  methods, string escaping, and whitespace/spacing logic (isGroupEnd).
  Estimated: 745 lines, ~200-400 spec fixes.

PRIORITY 4: PARSER — Selector/media/@supports structured parsing
  Raw text collection for selectors, media queries, and @supports conditions
  should be replaced with dart-sass's structured parsing methods.
  Estimated: 680 lines, ~100-200 spec fixes.

PRIORITY 5: EVALUATOR — CssVisitor + _ImportedCssVisitor
  Required for @import of modules that use @use/@forward. Currently stubbed
  with UnsupportedOperationException.
  Estimated: 330 lines, ~50-100 spec fixes.

PRIORITY 6: EXTEND — weaveParents full LCS + missing helper functions
  The simplified weaveParents causes incorrect selector output for complex
  combinator interactions.
  Estimated: 200 lines, ~30-50 spec fixes.

PRIORITY 7: MODULE — Configuration.throughForward + Module trait gaps
  Missing fields (upstream, variableNodes, extensionStore, cloneCss) and
  Configuration.throughForward block proper @forward...with propagation.
  Estimated: 250 lines, ~30-50 spec fixes.

PRIORITY 8: IMPORT — ImportCache + FilesystemImporter full port
  Basic functionality works but many resolution paths are simplified.
  Estimated: 400 lines, ~20-40 spec fixes.

 1a. ExtensionStore (extension_store.dart -> ExtensionStore.scala)
------------------------------------------------------------------------------

PORTED (faithful):
  - ExtensionStore trait (public API surface):
    isEmpty, simpleSelectors, extensionsWhereTarget, addSelector,
    addExtension, addExtensions, cloneStore, extensionsByExtender
  - EmptyExtensionStore singleton (all methods throw UnsupportedOperationException)
  - MutableExtensionStore (default mutable impl):
    All 6 state fields: _selectors, _extensions, _extensionsByExtender,
    _mediaContexts, _sourceSpecificity, _originals
  - Static helpers: extend(), replace(), _extendOrReplace()
  - Public API: addSelector, addExtension, addExtensions, cloneStore
  - Private pipeline:
    _registerSelector, _simpleSelectors,
    _extendExistingExtensions, _extendExistingSelectors,
    _extendList, _extendComplex, _extendCompound, _extendSimple,
    _extendPseudo,
    _extenderForSimple, _extenderForCompound, _sourceSpecificityFor,
    _unifyExtenders, _trim

SIMPLIFIED:
  - _trim(): Has a "// Avoid quadratic blowup" guard at >100 selectors,
    which matches dart-sass. However, ssg-sass skips the `rotateSlice`
    call for duplicate originals (just doesn't add the duplicate, which
    is functionally equivalent for correctness but may differ in ordering).
  - addExtensions(): Uses `asInstanceOf[MutableExtensionStore]` to access
    internal state of the other store, matching dart-sass's approach but
    without the safer private constructor pattern.

EXTRA (backward-compat adapters not in dart-sass):
  - astExtensions map, addExtensionAst(), extendList(), extendComplex(),
    weaveExtension(), substituteInComplexUnified()
    These are legacy AST-based extension adapters used by EvaluateVisitor
    until it is rewired to the dart addExtension+addSelector API.
    They should eventually be removed.

MISSING: None significant. All core methods are present.

==============================================================================
 TOP PRIORITY GAPS (blocking correct behavior)
==============================================================================

1. ExtendFunctions.weaveParents() - The simplified interleave approach will
   produce incorrect output for selectors involving :root, ID selectors
   requiring unification, and complex combinator groupings. The full LCS-
   based weave with _groupSelectors, _firstIfRootish, _mustUnify,
   _complexIsParentSuperselector, and longestCommonSubsequence is needed
   for spec compliance.

2. Module trait missing fields - upstream, extensionStore, cloneCss are
   needed by the evaluator for @use/@forward/@import semantics.
   variableNodes is needed for source-span-aware error messages.
   preModuleComments is needed for comment ordering in output.

3. Configuration.throughForward() - Required for @forward ... with()
   to work correctly. Without it, configured variables won't propagate
   through @forward chains.

4. ImportCache invalidation - clearImport/clearCanonicalize are needed
   for watch mode. The missing import() convenience method is needed
   by the evaluator.

5. StylesheetGraph reload/remove - Needed for file-watching / incremental
   compilation. The simplified graph only does cycle detection.

==============================================================================
 LOWER PRIORITY GAPS (correctness edge cases / performance)
==============================================================================

6. ForwardedView/ShadowedView missing equals/hashCode, cloneCss
7. ForwardedView uses .collect() instead of lazy map views (performance)
8. ImportCache lacks per-importer caching layer (performance)
9. FilesystemImporter JVM implementation not found at expected path
10. SassString.isSpecialVariable missing if() detection
11. Configuration.ExplicitConfiguration vs isImplicit boolean
12. Module.setVariable missing nodeWithSpan parameter
